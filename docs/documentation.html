Overview

Solar Neo provides a single toolchain for curating Fedora (DNF5) packages and Flatpak remotes. Whether you manage a homelab or a campus of desktops, the CLI keeps dependency state and release metadata synchronized across every workstation.

The documentation below highlights the most common workflows. For release specific change logs, head over to the version archive.

Installation

Download the newest bundle from the latest release page and follow the included installation guide. The archive contains the installer script and usage notes.

If you need a previous release, pick the appropriate build from the version archive.

Core commands

  • solar status — display enabled repositories, Flatpak remotes, and last sync timestamps.
  • solar install <package> — install a package across both DNF5 and Flatpak backends if available.
  • solar pin <package>=<version> — lock a package to a known-good build and prevent accidental upgrades.
  • solar diff <versionA> <versionB> — compare manifests to understand what changed between releases.

All commands support --help flags for additional switches and examples.

Configuration

Solar Neo reads configuration from /etc/solar/solar.toml. Common keys include:

  • channels — define the remote DNF5 and Flatpak endpoints to sync.
  • default_profile — pick the profile that loads on launch. Useful when testing staging vs. production settings.
  • allow_beta — set to true to automatically enroll machines into beta drop-ins.

Apply changes with sudo solar reload. The command validates syntax before replacing live configuration.

Automation tips

Solar Neo is script-friendly and can be embedded in provisioning pipelines.

  • Export SOLAR_NONINTERACTIVE=1 to suppress prompts during unattended installs.
  • Use solar plan to preview upcoming upgrades and emit machine-readable JSON for CI gates.
  • Check ~/.local/share/solar/logs for detailed execution logs and signature verification output.
Tip: combine solar plan with jq to produce dashboards that highlight risky updates before rollout.

Troubleshooting

  • Signature mismatch — rerun solar fetch --refresh to clear caches and download fresh metadata.
  • Flatpak remote offline — validate the remote URL with flatpak remote-ls and confirm firewall access.
  • Service stalled — check systemctl status solar.service to review journal entries and restart the daemon.

Still stuck? File an issue on GitHub with the output of solar report.
